# MVP 详细开发计划（8周）

## Sprint 1（第1-2周）：基础搭建 + 交易所API对接骨架

### 后端
- [ ] 项目初始化：FastAPI + SQLAlchemy + Alembic
- [ ] 数据库设计实现：users, data_assets, transactions, exchange_records 核心表
- [ ] 用户注册/登录 API（手机号 + 验证码）
- [ ] 实名认证接口（数据资产确权前置）
- [ ] JWT认证中间件
- [ ] 贵阳大数据交易所API对接骨架（封装SDK：认证、资产登记、交易、结算接口）
- [ ] 基础错误处理和日志

### 前端
- [ ] Next.js 项目初始化
- [ ] 基础页面框架和路由
- [ ] 登录/注册页面
- [ ] 实名认证页面
- [ ] 基础UI组件库选择和主题配置

### 基础设施
- [ ] 开发环境搭建（Docker Compose: PostgreSQL + Redis + ES）
- [ ] CI/CD 基础流水线
- [ ] 代码规范（linter、pre-commit hooks）
- [ ] 交易所沙箱环境联调账号配置

### 交付物
- 可注册登录的应用骨架（含实名认证流程）
- 数据库已迁移
- 交易所API SDK骨架（可调通沙箱认证接口）
- 开发环境文档

---

## Sprint 2（第3-4周）：数据资产上传与资产化处理管线

### 后端
- [ ] 健康数据上传API（支持体检报告、慢病数据等模板）
- [ ] 结构化模板定义（JSON Schema，3-5种健康数据模板）
- [ ] 自动化脱敏引擎（正则匹配：姓名/身份证/手机号/地址/医院名等）
- [ ] 数据质量评分（完整度、规范度、时效性）
- [ ] 数据资产包生成（脱敏数据 + 元数据 + 质量报告 → 标准资产包格式）
- [ ] 资产状态机（draft → desensitizing → packaged → pending_review → approved → registered → listed → sold）
- [ ] 草稿保存机制

### 前端
- [ ] 健康数据上传表单（基于模板的动态表单）
- [ ] 数据上传引导流程（上传 → 预览 → 确认脱敏 → 授权资产化）
- [ ] 草稿保存和续填
- [ ] 我的数据资产列表（含状态跟踪）
- [ ] 数据资产详情页（摘要预览/样本数据展示）

### 交付物
- 可以完整上传一份健康数据并生成资产包
- 脱敏规则初步生效
- 数据资产包格式符合交易所登记要求

---

## Sprint 3（第5-6周）：交易所登记 + 基础交易 + 区块链记录

### 后端
- [ ] 对接交易所资产登记接口（资产包 → 交易所审核 → 确权 → 上架）
- [ ] 对接交易所交易接口（挂牌 → 摘牌 → 成交确认）
- [ ] Elasticsearch 索引设计和数据同步
- [ ] 搜索 API（关键词 + 数据类型筛选 + 质量等级 + 分页）
- [ ] 微信支付集成（JSAPI/H5）
- [ ] 订单创建 → 支付 → 交易所成交确认 → 数据交付完整流程
- [ ] 区块链存证基础版（交易记录哈希上链）
- [ ] 购买后数据资产下载/访问授权逻辑

### 前端
- [ ] 首页：数据资产市场 + 搜索框
- [ ] 搜索结果页（筛选条件侧栏：数据类型、质量等级、价格区间）
- [ ] 数据资产详情页（摘要、样本预览、质量评分、交易所登记状态）
- [ ] 支付流程页面
- [ ] 已购买数据资产列表 & 下载

### 交付物
- 数据资产可登记到交易所并上架
- 搜索可用
- 支付 + 交易闭环跑通（沙箱环境）
- 交易记录上链可查

---

## Sprint 4（第7-8周）：收益分配 + 管理后台 + 上线

### 后端
- [ ] 收益分配逻辑实现（数据提供者60% / 平台30% / 交易所10%）
- [ ] 每周与交易所结算对账模块
- [ ] 用户收益钱包 & 提现申请
- [ ] 管理后台 API（数据资产审核、用户管理、交易管理）
- [ ] 交易所对账看板API
- [ ] 数据校验和安全加固
- [ ] 性能优化和压测
- [ ] 部署脚本和生产环境配置

### 前端
- [ ] 管理后台：数据资产审核列表 + 审核操作
- [ ] 管理后台：用户列表 + 实名认证审核
- [ ] 管理后台：交易管理 + 交易所结算看板
- [ ] 用户端：我的收益页面（收益明细、提现、区块链存证查询）
- [ ] 全站联调和Bug修复
- [ ] 响应式适配（移动端基础可用）

### 运营准备
- [ ] 种子用户邀请（目标50人数据提供者）
- [ ] 初始数据资产内容准备（运营准备标杆数据资产包）
- [ ] 与交易所确认正式上线流程 & 合规审核
- [ ] 用户协议和隐私政策定稿（含数据资产授权协议）
- [ ] 数据资产交易免责声明定稿

### 交付物
- MVP版本上线
- 50+种子用户 + 100+数据资产包
- 交易所正式登记 ≥ 50个资产
- 收益分配 & 周结算流程跑通
- 基本运营支撑就绪
