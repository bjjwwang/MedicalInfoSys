## 技术选型

### 后端

| 组件 | 选型 | 理由 |
|------|------|------|
| 主框架 | **Python FastAPI** | 开发效率高、AI生态好、async支持 |
| ORM | SQLAlchemy + Alembic | 成熟稳定，迁移管理方便 |
| 任务队列 | Celery + Redis | 异步任务（脱敏、审核、通知） |
| API文档 | FastAPI 自带 OpenAPI | 自动生成，前后端协作友好 |

> MVP阶段用单体 FastAPI 即可，等业务验证后再拆微服务。过早拆分只会增加复杂度。

### 前端

| 组件 | 选型 | 理由 |
|------|------|------|
| Web端 | **React + Next.js** | SEO友好（病例内容需要被搜索引擎收录）|
| 小程序 | **UniApp** | 一套代码覆盖微信/支付宝小程序 |
| 管理后台 | **Ant Design Pro** | 快速搭建，组件丰富 |
| 移动端(后期) | React Native 或 Flutter | 按需决定 |

### 数据层

| 组件 | 选型 | 用途 |
|------|------|------|
| 主数据库 | **PostgreSQL** | 用户、病例、交易等核心数据 |
| 缓存 | **Redis** | 会话、热门病例缓存、排行榜 |
| 搜索引擎 | **Elasticsearch** | 病例全文搜索、相似度检索 |
| 对象存储 | **阿里云 OSS / 腾讯 COS** | 影像资料、检查报告 |
| 消息队列 | **RabbitMQ / Redis Streams** | 服务间异步通信 |

### AI 能力

| 能力 | 实现方式 | 用途 |
|------|----------|------|
| 自动脱敏 | NER模型 (fine-tuned BERT) + 规则引擎 | 自动识别并遮蔽PII |
| 病例摘要 | LLM API (Claude/GPT) | 长病历自动摘要 |
| 相似匹配 | 向量检索 (Milvus/Qdrant) | 相似病例推荐 |
| 辅助录入 | LLM + 结构化提取 | 非结构化病历 → 结构化模板 |
| 内容审核 | LLM + 规则引擎 | 检测虚假、敏感、违规内容 |

### 基础设施

| 组件 | 选型 | 说明 |
|------|------|------|
| 云服务 | 阿里云 / 腾讯云 | 国内合规，医疗云资质 |
| 容器化 | Docker + Docker Compose | MVP阶段足够 |
| 编排(后期) | Kubernetes | 规模化后使用 |
| CI/CD | GitHub Actions / GitLab CI | 自动化构建部署 |
| 监控 | Prometheus + Grafana | 系统监控 |
| 日志 | ELK Stack | 日志收集分析 |
| APM | SkyWalking / Sentry | 性能监控、错误追踪 |

## 架构原则

1. **MVP先行**: 单体架构起步，验证后再拆分
2. **安全第一**: 数据加密传输（TLS）+ 静态加密 + 访问控制
3. **脱敏前置**: 任何数据入库前必须经过脱敏流水线
4. **审计可追溯**: 所有数据访问和操作留有审计日志
5. **弹性设计**: 核心服务降级策略，避免单点故障
