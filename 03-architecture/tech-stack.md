## 技术选型

### 后端

| 组件 | 选型 | 理由 |
|------|------|------|
| 主框架 | **Python FastAPI** | 开发效率高、AI生态好、async支持 |
| ORM | SQLAlchemy + Alembic | 成熟稳定，迁移管理方便 |
| 任务队列 | Celery + Redis | 异步任务（脱敏、资产加工、结算、通知） |
| API文档 | FastAPI 自带 OpenAPI | 自动生成，前后端协作友好 |
| 数据资产加工 | **Celery Pipeline + Pandas** | 多步骤流水线编排，数据清洗转换 |

> MVP阶段用单体 FastAPI 即可，等业务验证后再拆微服务。过早拆分只会增加复杂度。

### 前端

| 组件 | 选型 | 理由 |
|------|------|------|
| Web端 | **React + Next.js** | SEO友好（数据资产内容需要被搜索引擎收录）|
| 小程序 | **UniApp** | 一套代码覆盖微信/支付宝小程序 |
| 管理后台 | **Ant Design Pro** | 快速搭建，组件丰富，含数据资产管理面板 |
| 移动端(后期) | React Native 或 Flutter | 按需决定 |

### 数据层

| 组件 | 选型 | 用途 |
|------|------|------|
| 主数据库 | **PostgreSQL** | 用户、数据资产、交易等核心数据 |
| 缓存 | **Redis** | 会话、热门数据资产缓存、排行榜 |
| 搜索引擎 | **Elasticsearch** | 数据资产全文搜索、相似度检索 |
| 对象存储 | **阿里云 OSS / 腾讯 COS** | 原始健康数据文件、脱敏后资产包 |
| 消息队列 | **RabbitMQ / Redis Streams** | 服务间异步通信、资产加工事件驱动 |

### 区块链层

| 组件 | 选型 | 理由 |
|------|------|------|
| 区块链平台 | **长安链 ChainMaker** (首选) 或 **FISCO BCOS** | 国内政府认可的联盟链平台，合规性好，贵阳大数据交易所生态兼容 |
| 智能合约 | **Solidity (FISCO)** 或 **Go/Rust (长安链)** | 数据确权、访问记录上链、收益分配规则 |
| 链上功能 | 数据确权存证 + 访问追踪 + 收益分配 | 谁查看了什么数据、何时查看，触发收益分成 |
| SDK集成 | **ChainMaker Python SDK / FISCO Python SDK** | 与 FastAPI 后端无缝集成 |
| 节点部署 | **联盟链节点 (1-3个自有节点)** | 接入交易所联盟链网络 |

### 数据交易所对接

| 组件 | 选型 | 用途 |
|------|------|------|
| 交易所适配网关 | **FastAPI 独立服务** | 对接贵阳大数据交易所技术接口，协议转换 |
| 数据资产登记 | 交易所 API | 资产元数据注册、确权、上架 |
| 结算对接 | 交易所结算 API | 每周结算周期，批量对账 |
| 数据传输 | **HTTPS + 数据信封加密** | 脱敏后数据资产安全传输至交易所 |

### AI 能力

| 能力 | 实现方式 | 用途 |
|------|----------|------|
| 自动脱敏 | NER模型 (fine-tuned BERT) + 规则引擎 | 自动识别并遮蔽PII，健康数据脱敏 |
| 数据摘要 | LLM API (Claude/GPT) | 长健康记录自动摘要，资产描述生成 |
| 相似匹配 | 向量检索 (Milvus/Qdrant) | 相似数据资产推荐 |
| 辅助录入 | LLM + 结构化提取 | 非结构化健康数据 → 结构化模板 |
| 内容审核 | LLM + 规则引擎 | 检测虚假、敏感、违规内容 |
| 资产定价 | 统计模型 + 规则引擎 | 基于数据稀缺性、完整度、时效性自动建议定价 |

### 基础设施

| 组件 | 选型 | 说明 |
|------|------|------|
| 云服务 | 阿里云 / 腾讯云 | 国内合规，医疗云资质 |
| 容器化 | Docker + Docker Compose | MVP阶段足够 |
| 编排(后期) | Kubernetes | 规模化后使用 |
| CI/CD | GitHub Actions / GitLab CI | 自动化构建部署 |
| 监控 | Prometheus + Grafana | 系统监控，含区块链节点状态监控 |
| 日志 | ELK Stack | 日志收集分析 |
| APM | SkyWalking / Sentry | 性能监控、错误追踪 |

## 架构原则

1. **MVP先行**: 单体架构起步，验证后再拆分
2. **安全第一**: 数据加密传输（TLS）+ 静态加密 + 访问控制
3. **脱敏前置**: 任何数据入库前必须经过脱敏流水线
4. **审计可追溯**: 所有数据访问和操作留有审计日志，关键操作上链存证
5. **弹性设计**: 核心服务降级策略，避免单点故障
6. **数据资产合规**: 遵循《数据安全法》《个人信息保护法》及贵阳大数据交易所数据资产登记规范
7. **链上确权**: 数据资产权属通过区块链确权存证，确保数据提供方权益可追溯
8. **收益透明**: 区块链记录数据消费行为，收益分配规则公开透明，每周结算
