🎯 这个版本是做什么的
一句话：用 3 周、约 1.4 万，跑通"用户上传数据 → 别人用代币解锁下载 → 卖家获得代币收益"的完整闭环，验证产品流程与用户价值。 支付先用代币模拟，等真的验证出付费意愿再按阶梯升级到真实支付。

⚠ 这个版本明确不解决的问题
真实付费意愿验证（用代币，非真钱）、合规背书（暂不接交易所）、纠纷存证（无区块链）、规模化搜索、收益自动分配、AI脱敏。

代币的含义：用户和平台方在 MVP 期约定代币兑换规则（比如 admin 手动充值代币 / 新用户赠送），验证的是"愿意花"而非"愿意付真金白银"。真实付费验证要加支付升级包 P1（+¥800，下方有详细说明）。

如果当前诉求是"融资/招投标需要完整合规演示"，请看完整版 ¥40,000 / ¥58,000 方案。

📋 功能范围（保留 vs 砍掉）

✅ 保留（核心闭环必须）

  • 用户注册登录（手机号+短信）
  • 数据上传（结构化模板，5种常见类型）
  • 规则引擎脱敏（身份证/手机/姓名/地址）
  • 基础质量评分（完整性+时效性）
  • 数据市场浏览（列表+详情）
  • 关键词搜索（SQL LIKE，够用）
  • 代币余额 + 下单消费代币
  • 购买前样本预览 / 购买后解锁下载
  • 卖家代币收益 + 后台可查
  • 管理后台：数据审核、用户管理、代币充值
  • 响应式 Web（PC + 手机浏览器）

❌ 砍掉（非必须/后续再补）

  • 微信支付 / 支付宝真实支付
  • 贵阳大数据交易所API对接
  • 区块链节点部署 + 智能合约
  • 区块链溯源可视化
  • AI脱敏引擎（医疗BERT NER）
  • Elasticsearch 语义搜索
  • 收益分红 / 自动分账引擎
  • 代币 → 真钱提现
  • 小程序端 / Android 原生 App
  • 等保适配 / 渗透测试
  • 交易所对接仪表盘、结算管理
砍掉的逻辑：在商业假设未验证之前，这些功能都属于"假设验证后才值得投入"的范畴。 比如区块链存证——如果根本没几个用户、没几笔交易，上链没意义；再比如交易所对接——如果数据没人买，再合规也是空的。 所以先把钱花在"证明有人要"上。

🎯 怎么算"验证成功"

上线后 1-2 个月内，建议看以下几个核心指标决定下一步是升级到完整版还是调整方向。

维度 关键指标 🏆 强信号（升级V1.0） ⚠ 弱信号（需要调整）
供给 上传用户数 / 活跃上传率 ≥ 100 用户上传，周活 20%+ < 20 用户，活跃几乎为零
需求 代币消费笔数 / 询单转化 ≥ 20 笔代币消费 + 访谈时 ≥ 50% 用户说"如果是真钱也愿意花" 代币免费送也没人花
定价 代币价格分布 不同数据代币定价有明显分层 所有数据都压在最低档代币
客诉 纠纷率 / 数据质量投诉 < 5%，用户愿意复购 > 20%，质疑数据真实性
合规 监管/采购方要求 开始有机构询问交易所资质 完全是C端零散交易
验证路径建议：
  1. MVP 上线后 1-2 个月，跑代币模式 + 用户访谈，验证"愿不愿意花"
  2. 如果信号积极 → 加支付升级包 P1（人工收款，+¥800），真金白银验证
  3. 真实付费跑通 → 按需加 P2/P3（聚合支付/官方 API）
  4. 出现机构询问资质 → 加升级包 A（交易所对接）
  5. 出现纠纷存证需求 → 加升级包 B（区块链）
