模块一：用户服务

功能清单

1.1 注册/登录

  • 手机号 + 验证码注册登录
  • 微信/支付宝 OAuth 登录
  • 小程序静默登录
  • JWT Token 管理（access_token + refresh_token）

1.2 用户资料

  • 基本信息管理（昵称、头像、简介）
  • 角色选择（患者/医生/机构）
  • 个人主页（发布的案例、收到的评价）

1.3 医生认证

  • 认证流程：
  • 填写真实姓名、医院、科室、职称
  • 上传执业医师资格证照片
  • 上传工作证/胸牌（辅助验证）
  • 后台人工审核（对接国家医师注册查询系统校验）
  • 审核通过，获得认证标识
  • 认证等级：
  • 初级认证：基本信息 + 资格证
  • 高级认证：+ 医院官方确认函 / 学术成果

1.4 信用体系

  • 信用分维度：
  • 案例质量（同行评审得分）
  • 交易信誉（纠纷率、退款率）
  • 活跃贡献（上传频率、讨论参与度）
  • 信用分影响：
  • 高分：案例优先展示、更高提现额度
  • 低分：限制上传、降低曝光

1.5 钱包与提现

  • 余额管理
  • 充值（微信/支付宝）
  • 提现（银行卡，T+1到账）
  • 收支明细

API 设计（核心接口）

POST   /api/v1/auth/refresh            刷新token

PUT    /api/v1/users/me                更新用户信息
POST   /api/v1/users/certification     提交医生认证
GET    /api/v1/users/{id}/profile      获取用户公开资料

模块二：病例服务（核心模块）

功能清单

2.1 病例录入

  • 结构化模板录入
  • 预设模板：按病种提供不同的结构化模板
  • 字段引导：每个字段有示例和说明
  • 草稿保存：支持分多次填写
  • AI 辅助录入
  • 上传非结构化病历文档（PDF/图片）
  • AI 自动提取结构化信息，用户确认修正
  • 智能推荐 ICD 编码和标签
  • 影像/报告上传
  • 支持 CT、MRI、X光、病理报告、检验报告
  • 自动水印和脱敏处理

2.2 自动脱敏流水线

原始内容 → NER识别PII → 规则引擎补充 → 人工复核 → 脱敏完成

脱敏范围：
- 患者姓名 → 隐去
- 身份证号 → 隐去
- 手机号/地址 → 隐去
- 医院名称 → 可选保留/隐去
- 日期 → 模糊化（精确到月或相对时间）
- 影像中的文字信息 → OCR检测 + 遮蔽

2.3 内容审核

审核维度：
1. 脱敏完整性: PII是否完全清除
2. 医学合理性: 诊断和治疗方案是否合理
3. 内容真实性: 是否有明显杜撰/抄袭迹象
4. 合规性: 是否涉及违禁药品/非法疗法

审核流程：

AI自动审核 → 通过则发布 / 存疑则转人工 → 人工审核 → 通过/驳回/要求修改

2.4 病例展示

公开展示（免费）：
- 标题、疾病分类、患者基本信息（年龄段/性别）
- 主诉和症状摘要
- 治疗类型和疗效结果
- 评分和评价摘要

付费解锁内容：
- 完整诊断过程
- 详细治疗方案（用药/手术/中医细节）
- 影像资料和检查报告
- 随访记录

2.5 病例搜索

搜索维度：
- 关键词全文搜索
- 按疾病分类/ICD编码筛选
- 按症状组合筛选
- 按治疗类型筛选
- 按疗效结果筛选
- 按评分/价格/时间排序

高级搜索：
- AI 语义搜索（"60岁男性反复发热两周找不到原因"）
- 相似病例推荐（基于向量相似度）

2.6 病例质量分级

等级 标准 展示权益
S级 罕见病 + 治愈 + 完整随访 + 同行高分 首页推荐、专题收录
A级 信息完整 + 疗效确切 + 有随访 分类推荐
B级 基本完整，有参考价值 正常展示
C级 信息不够完整，但有部分参考 低优先展示

模块三：交易服务

功能清单

3.1 定价策略

  • 作者自主定价（平台建议价格范围）
  • 价格区间参考：
  • 常见病案例：¥9-29
  • 疑难病案例：¥29-99
  • 罕见病/重大突破：¥99-299
  • 平台可设置最高限价，防止漫天要价
  • 动态定价建议（根据病种稀缺度、同类案例数量）

3.2 购买流程

浏览摘要 → 决定购买 → 选择支付方式 → 支付 → 解锁完整内容
            ↓
        已购买则直接查看

3.3 支付集成

  • 微信支付（小程序支付 / H5支付 / Native支付）
  • 支付宝支付
  • 余额支付
  • 苹果IAP（如有iOS App，需走苹果支付，30%抽成）

3.4 分账规则

角色 分成比例 说明
案例作者 70-80% 核心贡献者
平台 20-30% 运营、审核、技术

VIP作者（高信用分/高产出）可提升至 85% 分成。

3.5 退款机制

  • 购买后24小时内可申请退款
  • 退款条件：
  • 内容与描述严重不符
  • 案例信息明显虚假
  • 重复购买
  • 退款流程：
    买方申请 → 平台审核 → 同意则原路退款 / 拒绝则通知原因 → 可申诉

3.6 订阅管理

  • 订阅套餐的开通/续费/取消
  • 到期前提醒
  • 自动续费（需用户明确授权）
  • 套餐升降级

3.7 结算与提现

  • 交易完成后T+7确认收入（给退款窗口期）
  • 最低提现金额：¥100
  • 提现到银行卡，T+1到账
  • 需实名认证后才可提现
  • 发票/税务：代扣代缴个人所得税（劳务报酬）

模块四：搜索与推荐服务

功能清单

4.1 基础搜索（Elasticsearch）

  • 全文关键词搜索（中文分词：IK Analyzer）
  • 多字段加权搜索（标题权重 > 疾病名 > 症状 > 正文）
  • 筛选条件组合：疾病分类、治疗类型、疗效、价格区间、评分
  • 排序：相关度 / 最新 / 最热 / 评分最高 / 价格
  • 搜索建议（输入联想）
  • 搜索高亮

4.2 语义搜索（向量检索）

  • 用户用自然语言描述症状或情况
  • LLM 将查询转为结构化意图 + 向量
  • Milvus/Qdrant 向量相似度检索
  • 结合 ES 关键词搜索做混合排序

示例查询：

"我爸65岁，糖尿病十年，最近脚趾发黑，医院说要截肢，有没有保住的案例？"

→ 系统识别：糖尿病足/坏疽，65岁男性，寻求保肢治疗方案

4.3 相似病例推荐

  • 基于当前查看病例，推荐相似案例
  • 相似度维度：疾病类型、症状组合、患者特征、治疗方式
  • "看了又看"协同过滤推荐

4.4 个性化推荐

  • 医生端：根据科室和擅长领域推荐新案例
  • 患者端：根据搜索历史和收藏推荐相关案例
  • 首页推荐流：热门 + 个性化混合

4.5 热门与趋势

  • 热搜病种排行
  • 近期热门案例
  • 新增稀缺病种提醒

技术实现

搜索流水线

用户查询
  ↓
查询理解（分词/纠错/同义词扩展/意图识别）
  ↓
并行检索：ES关键词 + 向量语义
  ↓
结果融合（RRF排序）
  ↓
业务过滤（权限/状态/付费）
  ↓
返回结果 + 推荐

模块五：AI 服务

功能清单

5.1 自动脱敏引擎

技术方案:
- 第一层：正则表达式（手机号、身份证、邮箱等固定格式）
- 第二层：NER模型（人名、地名、机构名等实体识别）
- 第三层：LLM复核（检查是否有遗漏的敏感信息）

模型选型:
- 基础NER：fine-tuned BERT (医疗领域语料)
- 复核：Claude/GPT API调用
- 影像OCR脱敏：PaddleOCR + 区域遮蔽

5.2 病历结构化提取

输入：非结构化病历文本/图片
输出：结构化JSON

原始文本:
"患者张某，男，58岁，因反复胸闷气短3个月入院。
既往有高血压病史10年，糖尿病5年..."

→ 提取结果:
{
  "patient_age_range": "51-60",
  "patient_gender": "male",
  "chief_complaint": "反复胸闷气短3个月",
  "symptoms": [
    {"name": "胸闷", "duration": "3个月", "severity": "反复发作"},
    {"name": "气短", "duration": "3个月"}
  ],
  "medical_history": ["高血压10年", "糖尿病5年"],
  "suggested_icd": "I50.9",
  "suggested_tags": ["心力衰竭", "高血压", "糖尿病"]
}

5.3 病例摘要生成

  • 自动为长病例生成 200 字以内的摘要
  • 摘要包含：核心诊断、治疗亮点、疗效概述
  • 用于搜索结果展示和 SEO

5.4 内容审核

AI审核维度：
1. 脱敏检查: 是否残留PII
2. 合理性检查: 诊断与症状是否逻辑一致
3. 原创性检查: 与已有案例的文本相似度
4. 合规检查: 是否涉及违禁内容
5. 质量评估: 信息完整度打分

输出：通过 / 需人工复核 / 驳回（附理由）

5.5 智能问答（后期）

  • 用户上传自己的病历，AI基于平台案例库给出参考建议
  • 明确标注"仅供参考，不构成诊疗建议"
  • 引用具体案例来源

模型部署策略

模型 部署方式 理由
NER脱敏模型 自部署 (GPU服务器) 敏感数据不出域
OCR模型 自部署 影像数据不出域
LLM (摘要/提取) API调用 (Claude) 性价比高，输入已脱敏
向量模型 自部署 (text2vec) 频繁调用，成本控制
内容审核 API + 自部署混合 平衡准确率和成本

安全原则: 脱敏前的原始数据绝不发送到外部API
