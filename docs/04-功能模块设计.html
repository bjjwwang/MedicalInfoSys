模块一：用户服务

功能清单

1.1 注册/登录

  • 手机号 + 验证码注册登录
  • 微信/支付宝 OAuth 登录
  • 小程序静默登录
  • JWT Token 管理（access_token + refresh_token）

1.2 用户资料

  • 基本信息管理（昵称、头像、简介）
  • 角色选择（患者/数据提供者/医生/药企/医疗机构/保险机构）
  • 区块链钱包地址绑定（用于数据确权和收益分配）
  • 个人主页（发布的数据资产、收到的评价）

1.3 数据提供者认证

  • 个人数据提供者认证：
    1. 实名认证（姓名 + 身份证）
    2. 绑定区块链钱包地址
    3. 签署数据授权协议
    4. 审核通过，获得数据提供者标识
  • 医生认证流程：
    1. 填写真实姓名、医院、科室、职称
    2. 上传执业医师资格证照片
    3. 上传工作证/胸牌（辅助验证）
    4. 后台人工审核（对接国家医师注册查询系统校验）
    5. 审核通过，获得认证标识
  • 机构认证（药企/医院/保险公司）：
    1. 上传营业执照及相关资质
    2. 法人信息验证
    3. 绑定企业区块链钱包地址
    4. 审核通过，获得机构认证标识

1.4 我的数据资产（数据资产组合视图）

  • 已上传的健康数据资产列表
  • 资产状态跟踪：草稿 / 脱敏中 / 待确权 / 已上架交易所 / 已下架
  • 各资产被浏览次数、被购买次数
  • 数据资产总估值

1.5 收益看板

  • 总收益概览（累计收益、本月收益、待结算收益）
  • 收益来源明细：
    • 数据资产直接交易收益
    • 数据分红收益（消费追溯分成）
  • 收益趋势图表
  • 区块链分账记录查询（链上可验证）

1.6 信用体系

  • 信用分维度：
    • 数据资产质量（完整度、验证状态、交易所评级）
    • 交易信誉（纠纷率、退款率）
    • 活跃贡献（上传频率、数据更新频率）
  • 信用分影响：
    • 高分：数据资产优先展示、更高收益分成比例
    • 低分：限制上传、降低曝光

1.7 钱包与提现

  • 平台余额管理
  • 区块链钱包余额查看
  • 充值（微信/支付宝）
  • 提现（银行卡，T+1到账；区块链钱包提取）
  • 收支明细（含链上交易记录）

模块二：数据资产服务（核心模块）

功能清单

2.1 健康数据上传

  • 结构化模板录入
    • 预设模板：按数据类型提供不同的结构化模板（健康指标、用药记录、饮食记录、治疗结果、肠道微生物数据等）
    • 字段引导：每个字段有示例和说明
    • 草稿保存：支持分多次填写
  • AI 辅助录入
    • 上传非结构化健康数据文档（PDF/图片/体检报告）
    • AI 自动提取结构化信息，用户确认修正
    • 智能推荐数据分类标签
  • 数据文件上传
    • 支持体检报告、检验报告、可穿戴设备数据、基因检测报告、肠道微生物检测报告
    • 支持 CT、MRI、X光等影像资料
    • 自动水印和脱敏处理

2.2 数据资产打包流水线

原始健康数据 → AI结构化提取 → 自动脱敏 → 质量评分 → 资产打包 → 提交交易所审核 → 确权登记 → 上架交易

打包流程：

  1. 结构化处理: 将原始数据转为标准化结构
  2. 脱敏处理: NER识别PII → 规则引擎补充 → 人工复核 → 脱敏完成
  3. 质量评分: 数据完整度、时间跨度、验证状态等多维度评分
  4. 资产打包: 生成数据资产描述、摘要、预览信息
  5. 交易所注册: 提交至贵阳大数据交易所进行合规审核
  6. 确权登记: 审核通过后完成数据确权，写入区块链

脱敏范围：

  • 患者姓名 → 隐去
  • 身份证号 → 隐去
  • 手机号/地址 → 隐去
  • 医院名称 → 可选保留/隐去
  • 日期 → 模糊化（精确到月或相对时间）
  • 影像中的文字信息 → OCR检测 + 遮蔽

2.3 交易所注册与确权

  • 自动生成交易所要求的数据资产描述文件
  • 提交贵阳大数据交易所审核
  • 审核状态跟踪（待审核/审核中/已通过/已驳回）
  • 确权完成后获得数据资产唯一标识
  • 确权信息上链存证

2.4 内容审核（交易所合规审核）

审核维度：

  1. 脱敏完整性: PII是否完全清除
  2. 数据真实性: 数据是否有明显伪造/篡改迹象
  3. 合规性: 是否符合交易所数据交易规范
  4. 数据质量: 信息完整度、可用性评估

审核流程：

AI自动审核 → 通过则提交交易所 / 存疑则转人工 → 人工审核 → 通过/驳回/要求修改 → 交易所终审

2.5 数据资产展示

公开展示（免费预览）：

  • 数据资产标题、数据类型分类
  • 数据提供者基本信息（年龄段/性别）
  • 健康状况概要、数据覆盖时间范围
  • 质量评分和交易所认证状态

付费解锁内容：

  • 完整健康数据详情
  • 详细用药记录、治疗方案
  • 检查检验报告、影像资料
  • 长期随访和疗效跟踪数据

2.6 数据资产质量分级

等级标准展示权益
S级罕见病数据 + 长期随访 + 多维度完整 + 已确权交易所首页推荐、专题收录
A级数据完整 + 已脱敏验证 + 已确权分类推荐
B级基本完整，有参考价值，待确权正常展示
C级信息不够完整，但有部分参考价值低优先展示

模块三：数据资产交易服务

功能清单

3.1 定价策略

  • 基于数据资产属性的定价模型：
    • 数据完整度（字段覆盖率、时间跨度）
    • 数据稀缺度（罕见病种、特殊治疗方案）
    • 验证状态（是否已通过交易所确权）
    • 数据维度丰富度（单一指标 vs 多维度健康数据）
  • 价格区间参考：
    • 基础健康指标数据：¥5-20
    • 完整诊疗过程数据：¥20-100
    • 罕见病/长期随访/多维度数据：¥100-500
    • 批量数据集（脱敏聚合）：协议定价
  • 平台可设置最高限价，防止漫天要价
  • 动态定价建议（根据数据稀缺度、同类资产数量、市场供需）

3.2 交易所交易流程

浏览数据资产预览 → 决定购买 → 通过交易所下单 → 支付 → 交易所确认 → 解锁完整数据
                    ↓
                已购买则直接查看

交易通道：

  • 交易所通道: 通过贵阳大数据交易所撮合交易，适用于已确权资产
  • 平台直购通道: 平台内小额数据资产直接购买（未上交易所的资产）

3.3 支付集成

  • 微信支付（小程序支付 / H5支付 / Native支付）
  • 支付宝支付
  • 余额支付
  • 交易所结算通道（大额/批量交易）
  • 苹果IAP（如有iOS App，需走苹果支付，30%抽成）

3.4 区块链追溯与收益分配

数据浏览追溯机制：

用户浏览数据资产 → 区块链记录浏览行为（资产ID、浏览者ID、时间戳）
        ↓
用户后续产生消费行为（购药、问诊、购买保险等）
        ↓
系统追溯该用户此前浏览过的数据资产
        ↓
将消费金额的一定比例分配给被浏览数据的提供者（数据分红）

收益分配规则：

角色分成来源说明
数据提供者（患者/个人）数据资产交易收入 60-70% + 数据分红核心数据贡献者
平台（数据资产服务商）交易服务费 15-20%资产打包、脱敏、技术服务
交易所交易手续费 5-10%撮合交易、确权服务
数据加工方（医生/机构）数据增值收入 5-10%数据标注、验证、解读增值

VIP数据提供者（高信用分/高质量数据）可提升至 75% 分成。

3.5 数据分红机制（消费追溯分成）

  • 区块链记录所有数据浏览行为
  • 当浏览者产生关联消费（购药、问诊、购买保险等）时触发分红
  • 分红计算：
    • 消费金额 × 分红比例（2-5%）
    • 按浏览时间衰减权重分配给所有被浏览数据的提供者
    • 单次消费分红上限设置
  • 分红结算周期：每周结算一次
  • 分红记录上链存证，可审计追溯

3.6 交易所订单管理

  • 交易所挂单/撤单
  • 订单状态同步（交易所 ↔ 平台）
  • 交易所交割确认
  • 批量数据集交易管理

3.7 结算与提现

  • 交易所结算：每周与贵阳大数据交易所进行一次结算对账
  • 平台内结算：交易完成后T+7确认收入（给退款窗口期）
  • 数据分红结算：每周计算并分配
  • 最低提现金额：¥50
  • 提现到银行卡/区块链钱包
  • 需实名认证后才可提现
  • 发票/税务：代扣代缴个人所得税（劳务报酬）

3.8 退款机制

  • 购买后24小时内可申请退款
  • 退款条件：
    • 数据内容与描述严重不符
    • 数据信息明显虚假或质量低于承诺
    • 重复购买
  • 退款流程： 
    买方申请 → 平台审核 → 同意则原路退款 / 拒绝则通知原因 → 可申诉 → 交易所介入仲裁

模块四：搜索与推荐服务

功能清单

4.1 基础搜索（Elasticsearch）

  • 全文关键词搜索（中文分词：IK Analyzer）
  • 多字段加权搜索（标题权重 > 健康状况 > 数据类型 > 正文描述）
  • 筛选条件组合：健康状况分类、数据类型、治疗方式、质量评级、价格区间、交易所认证状态
  • 排序：相关度 / 最新 / 最热 / 评分最高 / 价格
  • 搜索建议（输入联想）
  • 搜索高亮

4.2 语义搜索（向量检索）

  • 用户用自然语言描述健康状况或数据需求
  • LLM 将查询转为结构化意图 + 向量
  • Milvus/Qdrant 向量相似度检索
  • 结合 ES 关键词搜索做混合排序

示例查询：

"有没有糖尿病患者长期服用二甲双胍同时配合饮食控制的完整健康数据？包含血糖监测和肠道菌群检测的优先。"

→ 系统识别：糖尿病，二甲双胍用药，饮食干预，血糖监测+肠道微生物数据，长期跟踪数据

4.3 相似数据资产推荐

  • 基于当前查看的数据资产，推荐相似资产
  • 相似度维度：健康状况类型、数据覆盖维度、数据提供者特征、治疗方式
  • "看了又看"协同过滤推荐

4.4 交易所数据资产分类搜索

  • 按交易所数据资产类目体系浏览
  • 交易所认证资产专区搜索
  • 已确权资产筛选
  • 交易所热门资产同步展示

4.5 个性化推荐

  • 医生/研究者端：根据研究方向和关注领域推荐新数据资产
  • 药企端：根据药品管线推荐相关疾病领域数据
  • 保险机构端：根据险种推荐相关健康数据
  • 首页推荐流：热门 + 个性化混合

4.6 热门与趋势

  • 热搜健康数据类型排行
  • 近期热门数据资产
  • 新增稀缺数据资产提醒
  • 交易所热门交易趋势

模块五：AI 服务

功能清单

5.1 自动脱敏引擎

技术方案:

  • 第一层：正则表达式（手机号、身份证、邮箱等固定格式）
  • 第二层：NER模型（人名、地名、机构名等实体识别）
  • 第三层：LLM复核（检查是否有遗漏的敏感信息）

模型选型:

  • 基础NER：fine-tuned BERT (医疗领域语料)
  • 复核：Claude/GPT API调用
  • 影像OCR脱敏：PaddleOCR + 区域遮蔽

5.2 数据资产打包自动化

健康数据结构化提取：

输入：非结构化健康数据文本/图片/报告
输出：结构化JSON

原始文本:
"患者张某，男，58岁，因反复胸闷气短3个月入院。
既往有高血压病史10年，糖尿病5年。长期服用二甲双胍和厄贝沙坦。
肠道菌群检测显示双歧杆菌偏低..."

→ 提取结果:
{
  "patient_age_range": "51-60",
  "patient_gender": "male",
  "chief_complaint": "反复胸闷气短3个月",
  "symptoms": [
    {"name": "胸闷", "duration": "3个月", "severity": "反复发作"},
    {"name": "气短", "duration": "3个月"}
  ],
  "medical_history": ["高血压10年", "糖尿病5年"],
  "medications": ["二甲双胍", "厄贝沙坦"],
  "microbiome_data": {"bifidobacterium": "偏低"},
  "suggested_icd": "I50.9",
  "suggested_tags": ["心力衰竭", "高血压", "糖尿病", "肠道菌群"],
  "data_types": ["诊疗记录", "用药记录", "微生物检测"]
}

资产打包自动化流程：

  1. AI 提取结构化数据
  2. 自动生成数据资产描述和摘要
  3. 自动分类和标签推荐
  4. 生成交易所所需的数据资产说明文件
  5. 预估数据资产价值

5.3 数据质量评分（交易所合规）

  • 数据完整度评分：字段覆盖率、关键信息缺失检测
  • 数据时间跨度评分：长期跟踪数据加分
  • 数据一致性检查：多维度数据之间的逻辑一致性
  • 数据稀缺度评估：同类数据市场供给分析
  • 交易所合规性检查：是否满足交易所上架标准
  • 输出：质量评分（0-100）+ 合规报告 + 改进建议

5.4 智能匹配（健康画像匹配）

  • 基于健康数据画像匹配相似数据资产
  • 匹配维度：疾病类型、症状组合、年龄段、治疗方式、数据维度
  • 为数据需求方（药企、研究者、保险机构）推荐匹配的数据资产
  • 为数据提供者推荐可参考的同类数据定价

5.5 数据资产摘要生成

  • 自动为数据资产生成 200 字以内的摘要
  • 摘要包含：核心健康状况、数据覆盖维度、数据亮点
  • 用于搜索结果展示和交易所资产描述

5.6 内容审核（交易所合规审核）

AI审核维度：

  1. 脱敏检查: 是否残留PII
  2. 数据真实性检查: 数据是否逻辑一致、无明显伪造
  3. 原创性检查: 与已有数据资产的相似度检测
  4. 合规检查: 是否符合交易所数据交易规范
  5. 质量评估: 数据完整度和可用性打分

输出：通过 / 需人工复核 / 驳回（附理由和合规改进建议）

5.7 智能问答（后期）

  • 用户上传自己的健康数据，AI基于平台数据资产库给出参考建议
  • 明确标注"仅供参考，不构成诊疗建议"
  • 引用具体数据资产来源

模型部署策略

模型部署方式理由
NER脱敏模型自部署 (GPU服务器)敏感数据不出域
OCR模型自部署影像数据不出域
LLM (摘要/提取/打包)API调用 (Claude)性价比高，输入已脱敏
向量模型自部署 (text2vec)频繁调用，成本控制
内容审核/合规检查API + 自部署混合平衡准确率和成本
质量评分模型自部署频繁调用，需定制化

安全原则: 脱敏前的原始数据绝不发送到外部API
