# 模块三：数据资产交易服务

## 功能清单

### 3.1 定价策略
- 基于数据资产属性的定价模型：
  - 数据完整度（字段覆盖率、时间跨度）
  - 数据稀缺度（罕见病种、特殊治疗方案）
  - 验证状态（是否已通过交易所确权）
  - 数据维度丰富度（单一指标 vs 多维度健康数据）
- 价格区间参考：
  - 基础健康指标数据：¥5-20
  - 完整诊疗过程数据：¥20-100
  - 罕见病/长期随访/多维度数据：¥100-500
  - 批量数据集（脱敏聚合）：协议定价
- 平台可设置最高限价，防止漫天要价
- 动态定价建议（根据数据稀缺度、同类资产数量、市场供需）

### 3.2 交易所交易流程

```
浏览数据资产预览 → 决定购买 → 通过交易所下单 → 支付 → 交易所确认 → 解锁完整数据
                    ↓
                已购买则直接查看
```

交易通道：
- **交易所通道**: 通过贵阳大数据交易所撮合交易，适用于已确权资产
- **平台直购通道**: 平台内小额数据资产直接购买（未上交易所的资产）

### 3.3 支付集成
- 微信支付（小程序支付 / H5支付 / Native支付）
- 支付宝支付
- 余额支付
- 交易所结算通道（大额/批量交易）
- 苹果IAP（如有iOS App，需走苹果支付，30%抽成）

### 3.4 区块链追溯与收益分配

**数据浏览追溯机制：**
```
用户浏览数据资产 → 区块链记录浏览行为（资产ID、浏览者ID、时间戳）
        ↓
用户后续产生消费行为（购药、问诊、购买保险等）
        ↓
系统追溯该用户此前浏览过的数据资产
        ↓
将消费金额的一定比例分配给被浏览数据的提供者（数据分红）
```

**收益分配规则：**

| 角色 | 分成来源 | 说明 |
|------|---------|------|
| 数据提供者（患者/个人） | 数据资产交易收入 60-70% + 数据分红 | 核心数据贡献者 |
| 平台（数据资产服务商） | 交易服务费 15-20% | 资产打包、脱敏、技术服务 |
| 交易所 | 交易手续费 5-10% | 撮合交易、确权服务 |
| 数据加工方（医生/机构） | 数据增值收入 5-10% | 数据标注、验证、解读增值 |

VIP数据提供者（高信用分/高质量数据）可提升至 75% 分成。

### 3.5 数据分红机制（消费追溯分成）
- 区块链记录所有数据浏览行为
- 当浏览者产生关联消费（购药、问诊、购买保险等）时触发分红
- 分红计算：
  - 消费金额 × 分红比例（2-5%）
  - 按浏览时间衰减权重分配给所有被浏览数据的提供者
  - 单次消费分红上限设置
- 分红结算周期：每周结算一次
- 分红记录上链存证，可审计追溯

### 3.6 交易所订单管理
- 交易所挂单/撤单
- 订单状态同步（交易所 ↔ 平台）
- 交易所交割确认
- 批量数据集交易管理

### 3.7 结算与提现
- 交易所结算：每周与贵阳大数据交易所进行一次结算对账
- 平台内结算：交易完成后T+7确认收入（给退款窗口期）
- 数据分红结算：每周计算并分配
- 最低提现金额：¥50
- 提现到银行卡/区块链钱包
- 需实名认证后才可提现
- 发票/税务：代扣代缴个人所得税（劳务报酬）

### 3.8 退款机制
- 购买后24小时内可申请退款
- 退款条件：
  - 数据内容与描述严重不符
  - 数据信息明显虚假或质量低于承诺
  - 重复购买
- 退款流程：
  ```
  买方申请 → 平台审核 → 同意则原路退款 / 拒绝则通知原因 → 可申诉 → 交易所介入仲裁
  ```

## API 设计

```
POST   /api/v1/orders                        创建订单（平台直购）
POST   /api/v1/orders/exchange                创建交易所订单
POST   /api/v1/orders/{id}/pay               发起支付
GET    /api/v1/orders/{id}                   查询订单状态
POST   /api/v1/orders/{id}/refund            申请退款
GET    /api/v1/orders/me                     我的订单列表
GET    /api/v1/exchange/orders               交易所订单列表
GET    /api/v1/exchange/orders/{id}/status    交易所订单状态同步
POST   /api/v1/exchange/settlement/weekly    触发每周结算对账
GET    /api/v1/income/summary                收入概览（含交易收入+分红收入）
GET    /api/v1/income/details                收入明细
GET    /api/v1/income/dividends              数据分红明细
GET    /api/v1/income/chain-records          链上分配记录
POST   /api/v1/withdrawals                   申请提现
GET    /api/v1/dividends/trace/{consumer_id} 消费追溯分红详情
```

## 支付与交易安全
- 所有支付操作幂等处理（防止重复支付）
- 支付回调签名验证
- 对账机制：每日与支付渠道对账，每周与交易所对账
- 资金流水完整记录，支持审计
- 区块链存证：关键交易记录上链，不可篡改
- 数据浏览记录上链：确保分红追溯的公正透明
