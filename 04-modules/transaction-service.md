# 模块三：交易服务

## 功能清单

### 3.1 定价策略
- 作者自主定价（平台建议价格范围）
- 价格区间参考：
  - 常见病案例：¥9-29
  - 疑难病案例：¥29-99
  - 罕见病/重大突破：¥99-299
- 平台可设置最高限价，防止漫天要价
- 动态定价建议（根据病种稀缺度、同类案例数量）

### 3.2 购买流程

```
浏览摘要 → 决定购买 → 选择支付方式 → 支付 → 解锁完整内容
            ↓
        已购买则直接查看
```

### 3.3 支付集成
- 微信支付（小程序支付 / H5支付 / Native支付）
- 支付宝支付
- 余额支付
- 苹果IAP（如有iOS App，需走苹果支付，30%抽成）

### 3.4 分账规则

| 角色 | 分成比例 | 说明 |
|------|---------|------|
| 案例作者 | 70-80% | 核心贡献者 |
| 平台 | 20-30% | 运营、审核、技术 |

VIP作者（高信用分/高产出）可提升至 85% 分成。

### 3.5 退款机制
- 购买后24小时内可申请退款
- 退款条件：
  - 内容与描述严重不符
  - 案例信息明显虚假
  - 重复购买
- 退款流程：
  ```
  买方申请 → 平台审核 → 同意则原路退款 / 拒绝则通知原因 → 可申诉
  ```

### 3.6 订阅管理
- 订阅套餐的开通/续费/取消
- 到期前提醒
- 自动续费（需用户明确授权）
- 套餐升降级

### 3.7 结算与提现
- 交易完成后T+7确认收入（给退款窗口期）
- 最低提现金额：¥100
- 提现到银行卡，T+1到账
- 需实名认证后才可提现
- 发票/税务：代扣代缴个人所得税（劳务报酬）

## API 设计

```
POST   /api/v1/orders                  创建订单
POST   /api/v1/orders/{id}/pay         发起支付
GET    /api/v1/orders/{id}             查询订单状态
POST   /api/v1/orders/{id}/refund      申请退款
GET    /api/v1/orders/me               我的订单列表
POST   /api/v1/subscriptions           开通订阅
PUT    /api/v1/subscriptions/{id}      升降级订阅
DELETE /api/v1/subscriptions/{id}      取消订阅
GET    /api/v1/income/summary          收入概览
GET    /api/v1/income/details          收入明细
POST   /api/v1/withdrawals             申请提现
```

## 支付安全
- 所有支付操作幂等处理（防止重复支付）
- 支付回调签名验证
- 对账机制：每日与支付渠道对账
- 资金流水完整记录，支持审计
