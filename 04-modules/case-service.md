# 模块二：数据资产服务（核心模块）

## 功能清单

### 2.1 健康数据上传
- **结构化模板录入**
  - 预设模板：按数据类型提供不同的结构化模板（健康指标、用药记录、饮食记录、治疗结果、肠道微生物数据等）
  - 字段引导：每个字段有示例和说明
  - 草稿保存：支持分多次填写
- **AI 辅助录入**
  - 上传非结构化健康数据文档（PDF/图片/体检报告）
  - AI 自动提取结构化信息，用户确认修正
  - 智能推荐数据分类标签
- **数据文件上传**
  - 支持体检报告、检验报告、可穿戴设备数据、基因检测报告、肠道微生物检测报告
  - 支持 CT、MRI、X光等影像资料
  - 自动水印和脱敏处理

### 2.2 数据资产打包流水线

```
原始健康数据 → AI结构化提取 → 自动脱敏 → 质量评分 → 资产打包 → 提交交易所审核 → 确权登记 → 上架交易
```

打包流程：
1. **结构化处理**: 将原始数据转为标准化结构
2. **脱敏处理**: NER识别PII → 规则引擎补充 → 人工复核 → 脱敏完成
3. **质量评分**: 数据完整度、时间跨度、验证状态等多维度评分
4. **资产打包**: 生成数据资产描述、摘要、预览信息
5. **交易所注册**: 提交至贵阳大数据交易所进行合规审核
6. **确权登记**: 审核通过后完成数据确权，写入区块链

脱敏范围：
- 患者姓名 → 隐去
- 身份证号 → 隐去
- 手机号/地址 → 隐去
- 医院名称 → 可选保留/隐去
- 日期 → 模糊化（精确到月或相对时间）
- 影像中的文字信息 → OCR检测 + 遮蔽

### 2.3 交易所注册与确权
- 自动生成交易所要求的数据资产描述文件
- 提交贵阳大数据交易所审核
- 审核状态跟踪（待审核/审核中/已通过/已驳回）
- 确权完成后获得数据资产唯一标识
- 确权信息上链存证

### 2.4 内容审核（交易所合规审核）

审核维度：
1. **脱敏完整性**: PII是否完全清除
2. **数据真实性**: 数据是否有明显伪造/篡改迹象
3. **合规性**: 是否符合交易所数据交易规范
4. **数据质量**: 信息完整度、可用性评估

审核流程：
```
AI自动审核 → 通过则提交交易所 / 存疑则转人工 → 人工审核 → 通过/驳回/要求修改 → 交易所终审
```

### 2.5 数据资产展示

公开展示（免费预览）：
- 数据资产标题、数据类型分类
- 数据提供者基本信息（年龄段/性别）
- 健康状况概要、数据覆盖时间范围
- 质量评分和交易所认证状态

付费解锁内容：
- 完整健康数据详情
- 详细用药记录、治疗方案
- 检查检验报告、影像资料
- 长期随访和疗效跟踪数据

### 2.6 数据资产质量分级

| 等级 | 标准 | 展示权益 |
|------|------|---------|
| S级 | 罕见病数据 + 长期随访 + 多维度完整 + 已确权 | 交易所首页推荐、专题收录 |
| A级 | 数据完整 + 已脱敏验证 + 已确权 | 分类推荐 |
| B级 | 基本完整，有参考价值，待确权 | 正常展示 |
| C级 | 信息不够完整，但有部分参考价值 | 低优先展示 |

## API 设计（核心接口）

```
POST   /api/v1/data-assets                    创建数据资产（草稿）
PUT    /api/v1/data-assets/{id}               更新数据资产
POST   /api/v1/data-assets/{id}/package       触发资产打包流水线
POST   /api/v1/data-assets/{id}/submit        提交交易所审核
GET    /api/v1/data-assets/{id}               获取数据资产详情（根据权限返回不同内容）
GET    /api/v1/data-assets/{id}/full          获取完整数据（需购买或资产所有者）
DELETE /api/v1/data-assets/{id}               删除数据资产
GET    /api/v1/data-assets                    数据资产列表（支持筛选排序分页）
GET    /api/v1/data-assets/search             搜索数据资产
GET    /api/v1/data-assets/{id}/similar       获取相似数据资产
POST   /api/v1/data-assets/{id}/media         上传数据附件
POST   /api/v1/data-assets/ai-extract         AI提取结构化信息
GET    /api/v1/data-assets/me                 我的数据资产列表
GET    /api/v1/data-assets/{id}/exchange-status 查询交易所审核/确权状态
```
