# 模块一：用户服务

## 功能清单

### 1.1 注册/登录
- 手机号 + 验证码注册登录
- 微信/支付宝 OAuth 登录
- 小程序静默登录
- JWT Token 管理（access_token + refresh_token）

### 1.2 用户资料
- 基本信息管理（昵称、头像、简介）
- 角色选择（患者/数据提供者/医生/药企/医疗机构/保险机构）
- 区块链钱包地址绑定（用于数据确权和收益分配）
- 个人主页（发布的数据资产、收到的评价）

### 1.3 数据提供者认证
- 个人数据提供者认证：
  1. 实名认证（姓名 + 身份证）
  2. 绑定区块链钱包地址
  3. 签署数据授权协议
  4. 审核通过，获得数据提供者标识
- 医生认证流程：
  1. 填写真实姓名、医院、科室、职称
  2. 上传执业医师资格证照片
  3. 上传工作证/胸牌（辅助验证）
  4. 后台人工审核（对接国家医师注册查询系统校验）
  5. 审核通过，获得认证标识
- 机构认证（药企/医院/保险公司）：
  1. 上传营业执照及相关资质
  2. 法人信息验证
  3. 绑定企业区块链钱包地址
  4. 审核通过，获得机构认证标识

### 1.4 我的数据资产（数据资产组合视图）
- 已上传的健康数据资产列表
- 资产状态跟踪：草稿 / 脱敏中 / 待确权 / 已上架交易所 / 已下架
- 各资产被浏览次数、被购买次数
- 数据资产总估值

### 1.5 收益看板
- 总收益概览（累计收益、本月收益、待结算收益）
- 收益来源明细：
  - 数据资产直接交易收益
  - 数据分红收益（消费追溯分成）
- 收益趋势图表
- 区块链分账记录查询（链上可验证）

### 1.6 信用体系
- 信用分维度：
  - 数据资产质量（完整度、验证状态、交易所评级）
  - 交易信誉（纠纷率、退款率）
  - 活跃贡献（上传频率、数据更新频率）
- 信用分影响：
  - 高分：数据资产优先展示、更高收益分成比例
  - 低分：限制上传、降低曝光

### 1.7 钱包与提现
- 平台余额管理
- 区块链钱包余额查看
- 充值（微信/支付宝）
- 提现（银行卡，T+1到账；区块链钱包提取）
- 收支明细（含链上交易记录）

## API 设计（核心接口）

```
POST   /api/v1/auth/sms-code          发送验证码
POST   /api/v1/auth/login              登录
POST   /api/v1/auth/refresh            刷新token
GET    /api/v1/users/me                获取当前用户信息
PUT    /api/v1/users/me                更新用户信息
POST   /api/v1/users/certification     提交认证（数据提供者/医生/机构）
GET    /api/v1/users/{id}/profile      获取用户公开资料
PUT    /api/v1/users/me/wallet-bindng  绑定区块链钱包地址
GET    /api/v1/users/me/wallet         获取钱包信息（含链上余额）
POST   /api/v1/users/me/withdraw       申请提现
GET    /api/v1/users/me/assets         我的数据资产列表
GET    /api/v1/users/me/assets/summary 数据资产组合概览
GET    /api/v1/users/me/revenue        收益看板数据
GET    /api/v1/users/me/revenue/details 收益明细（含分红记录）
GET    /api/v1/users/me/revenue/chain-records 链上分账记录
```
