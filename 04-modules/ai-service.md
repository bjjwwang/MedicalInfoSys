# 模块五：AI 服务

## 功能清单

### 5.1 自动脱敏引擎

**技术方案:**
- 第一层：正则表达式（手机号、身份证、邮箱等固定格式）
- 第二层：NER模型（人名、地名、机构名等实体识别）
- 第三层：LLM复核（检查是否有遗漏的敏感信息）

**模型选型:**
- 基础NER：fine-tuned BERT (医疗领域语料)
- 复核：Claude/GPT API调用
- 影像OCR脱敏：PaddleOCR + 区域遮蔽

### 5.2 数据资产打包自动化

**健康数据结构化提取：**

输入：非结构化健康数据文本/图片/报告
输出：结构化JSON

```
原始文本:
"患者张某，男，58岁，因反复胸闷气短3个月入院。
既往有高血压病史10年，糖尿病5年。长期服用二甲双胍和厄贝沙坦。
肠道菌群检测显示双歧杆菌偏低..."

→ 提取结果:
{
  "patient_age_range": "51-60",
  "patient_gender": "male",
  "chief_complaint": "反复胸闷气短3个月",
  "symptoms": [
    {"name": "胸闷", "duration": "3个月", "severity": "反复发作"},
    {"name": "气短", "duration": "3个月"}
  ],
  "medical_history": ["高血压10年", "糖尿病5年"],
  "medications": ["二甲双胍", "厄贝沙坦"],
  "microbiome_data": {"bifidobacterium": "偏低"},
  "suggested_icd": "I50.9",
  "suggested_tags": ["心力衰竭", "高血压", "糖尿病", "肠道菌群"],
  "data_types": ["诊疗记录", "用药记录", "微生物检测"]
}
```

**资产打包自动化流程：**
1. AI 提取结构化数据
2. 自动生成数据资产描述和摘要
3. 自动分类和标签推荐
4. 生成交易所所需的数据资产说明文件
5. 预估数据资产价值

### 5.3 数据质量评分（交易所合规）
- 数据完整度评分：字段覆盖率、关键信息缺失检测
- 数据时间跨度评分：长期跟踪数据加分
- 数据一致性检查：多维度数据之间的逻辑一致性
- 数据稀缺度评估：同类数据市场供给分析
- 交易所合规性检查：是否满足交易所上架标准
- 输出：质量评分（0-100）+ 合规报告 + 改进建议

### 5.4 智能匹配（健康画像匹配）
- 基于健康数据画像匹配相似数据资产
- 匹配维度：疾病类型、症状组合、年龄段、治疗方式、数据维度
- 为数据需求方（药企、研究者、保险机构）推荐匹配的数据资产
- 为数据提供者推荐可参考的同类数据定价

### 5.5 数据资产摘要生成
- 自动为数据资产生成 200 字以内的摘要
- 摘要包含：核心健康状况、数据覆盖维度、数据亮点
- 用于搜索结果展示和交易所资产描述

### 5.6 内容审核（交易所合规审核）

AI审核维度：
1. **脱敏检查**: 是否残留PII
2. **数据真实性检查**: 数据是否逻辑一致、无明显伪造
3. **原创性检查**: 与已有数据资产的相似度检测
4. **合规检查**: 是否符合交易所数据交易规范
5. **质量评估**: 数据完整度和可用性打分

输出：通过 / 需人工复核 / 驳回（附理由和合规改进建议）

### 5.7 智能问答（后期）
- 用户上传自己的健康数据，AI基于平台数据资产库给出参考建议
- 明确标注"仅供参考，不构成诊疗建议"
- 引用具体数据资产来源

## 模型部署策略

| 模型 | 部署方式 | 理由 |
|------|---------|------|
| NER脱敏模型 | 自部署 (GPU服务器) | 敏感数据不出域 |
| OCR模型 | 自部署 | 影像数据不出域 |
| LLM (摘要/提取/打包) | API调用 (Claude) | 性价比高，输入已脱敏 |
| 向量模型 | 自部署 (text2vec) | 频繁调用，成本控制 |
| 内容审核/合规检查 | API + 自部署混合 | 平衡准确率和成本 |
| 质量评分模型 | 自部署 | 频繁调用，需定制化 |

**安全原则: 脱敏前的原始数据绝不发送到外部API**
