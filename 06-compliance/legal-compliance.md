## 二、数据合规

### 适用法律法规
- 《个人信息保护法》(PIPL)
- 《数据安全法》
- 《网络安全法》
- 《医疗卫生机构信息公开办法》
- 《人类遗传资源管理条例》（如涉及基因数据）
- 《健康医疗大数据标准、安全和服务管理办法》

### 数据分类分级

| 数据类别 | 安全级别 | 处理要求 |
|----------|---------|---------|
| 患者原始病历 | L4-极敏感 | 加密存储、脱敏后使用、不出境 |
| 医生身份信息 | L3-敏感 | 加密存储、最小范围使用 |
| 用户手机号/邮箱 | L3-敏感 | 加密存储、脱敏展示 |
| 脱敏后病例数据 | L2-内部 | 可用于AI训练、数据分析 |
| 公开展示信息 | L1-公开 | 正常使用 |
| 交易记录 | L2-内部 | 保留至少5年 |

### 隐私设计原则 (Privacy by Design)
1. **最小必要**: 只收集业务必需的数据
2. **目的限制**: 数据仅用于声明的目的
3. **存储限制**: 设定数据保留期限，过期删除
4. **单独同意**: 敏感个人信息需用户单独同意
5. **可撤回**: 用户可随时撤回同意、删除数据
6. **可携带**: 支持用户导出自己的数据

### 同意授权设计

病例上传需获取的授权：
```
□ 我确认此病例信息已经过充分脱敏，不包含可识别患者身份的信息
□ 我已获得患者（或其法定代理人）的知情同意，允许将脱敏后的诊疗信息用于医学知识交流
□ 我同意将此病例信息在平台上展示和交易，并理解平台的《内容使用协议》
□ 我理解并同意平台的《隐私政策》和《用户服务协议》
```

## 三、平台免责设计

### 核心免责声明

```
重要声明：
1. 本平台提供的所有病例信息仅供医学参考和学术交流使用，不构成任何诊疗建议。
2. 任何治疗决策应由具有执业资质的医生在充分了解患者具体情况后做出。
3. 患者不应根据平台上的案例信息自行诊断或治疗。
4. 平台不对因参考病例信息而产生的任何后果承担责任。
5. 每位患者的情况各不相同，相同疾病的治疗方案可能因个体差异而完全不同。
```

### 需要准备的法律文件
1. **用户服务协议** - 明确平台性质、用户权利义务
2. **隐私政策** - 数据收集、使用、存储、共享说明
3. **内容使用协议** - 病例版权、使用范围、责任划分
4. **医疗信息免责声明** - 上述核心免责
5. **知识产权协议** - 案例著作权归属
6. **退款政策** - 明确退款条件和流程

## 四、知识产权

### 病例版权归属
- **原则**: 病例著作权归作者（上传医生）所有
- **平台获得**: 非独占的展示和传播许可
- **买方获得**: 个人参考使用权，不得转售或公开传播
- **转载/引用**: 需注明来源和作者

### 反抄袭机制
- 新上传案例与已有案例进行相似度检测
- 疑似抄袭则人工复核
- 确认抄袭则下架 + 信用扣分

## 五、风险应对预案

| 风险场景 | 应对方案 |
|----------|---------|
| 数据泄露 | 72小时内报告主管部门、通知受影响用户、启动应急响应 |
| 虚假病例导致患者受害 | 配合调查、提供交易和审核记录、启动保险理赔 |
| 监管政策变化 | 保持与行业协会沟通、预留合规调整buffer |
| 知识产权纠纷 | 先下架争议内容、走争议解决流程 |
| 患者隐私投诉 | 立即核查脱敏情况、必要时下架、补偿 |

### 建议购买的保险
- 网络安全/数据泄露责任险
- 职业责任险
- 平台运营责任险
